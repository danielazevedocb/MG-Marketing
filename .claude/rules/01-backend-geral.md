---
description: Regras gerais para backend
alwaysApply: true
---

- Escreva código backend modular, previsível e fácil de manter.
- Prefira responsabilidades bem separadas entre camadas.
- Evite colocar muita lógica em controllers.
- Sempre valide entradas de dados.
- Sempre trate erros de forma consistente e clara.
- Considere segurança, autenticação, autorização e integridade dos dados ao propor mudanças.
- Prefira respostas de API consistentes e fáceis de consumir.
- Ao alterar fluxos de negócio, considere impacto em serviços, banco, DTOs e contratos da API.
- Mantenha o código preparado para crescimento sem criar complexidade desnecessária.
